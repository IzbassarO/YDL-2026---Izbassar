# 🎓 Грант-бот фонда Шахмардана Есенова (RAG + Streamlit)

Финальный проект 1-й недели **Yessenov Data Lab 2026**. Streamlit-чатбот на
`gemma4`, который отвечает на вопросы про гранты, программы и деятельность
фонда Есенова — **строго по данным с сайта `yessenovfoundation.org`**.

> **Главное правило:** бот не выдумывает. Если в собранных материалах нет
> ответа — он честно говорит «не нашёл», а не сочиняет дедлайны/суммы/условия.
> Это обеспечивается порогом релевантности RAG (ответ «не знаю» выдаётся
> **без вызова LLM**) + строгим системным промптом.

## Архитектура

```
scrape.py        сайт фонда → обход домена → data/corpus.jsonl (+ процедурные карточки)
build_index.py   corpus → чанкинг (700/120) → эмбеддинги text-1024 → FAISS (косинус)
rag.py           вопрос → эмбеддинг → FAISS top-4 → порог → gemma4 → ответ
app.py           Streamlit UI: чат, бейдж grounded/«не знаю», источники, лимиты, email
email_report.py  MailerSend: саммари диалога → izok2004@gmail.com (только по кнопке)
eval_rag.py      системная проверка: in-scope / out-of-scope / калибровка / email
config.py        ключи, пути, пороги, лимиты
```

## Запуск

```bash
pip install -r requirements.txt

python scrape.py         # сайт → data/corpus.jsonl
python build_index.py    # → data/index.faiss + data/meta.json
python eval_rag.py       # проверка RAG (без отправки email)
python eval_rag.py --email   # + реально отправит тестовое письмо себе

streamlit run app.py     # веб-интерфейс
```

> Векторная база (`data/`) собрана на сайте фонда. Её состав и статистика —
> в [`VECTOR_DB.md`](VECTOR_DB.md). Лог системной проверки бота — в
> [`TEST_REPORT.md`](TEST_REPORT.md).

## Калибровка `RELEVANCE_THRESHOLD`

Порог в `config.py` решает, считать ли лучший найденный чанк достаточно
релевантным. Слишком низкий → бот отвечает на чужие вопросы (галлюцинации),
слишком высокий → отказывается отвечать на свои.

Как калибровать:

1. Запусти `python eval_rag.py`.
2. Скрипт печатает `top_score` для группы **in-scope** (ответ есть в данных)
   и **out-of-scope** (нет в данных) и предлагает порог ровно между ними.
3. Поставь полученное значение в `RELEVANCE_THRESHOLD` (`config.py`).
4. Перепроверь: in-scope должны отвечать, out-of-scope — отказывать.

## Безопасность ключей

Ключи API лежат в `config.py` со значениями по умолчанию (учебный проект).
Для портфолио/публикации вынеси их в `.env` (переменные читаются через
`os.getenv`, см. `config.py`) — `.env` уже в `.gitignore`.

## Правила email (не нарушать)
