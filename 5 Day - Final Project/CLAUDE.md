# CLAUDE.md — RAG-чатбот по грантам фонда Есенова (YDL 2026, День 5)

> Этот файл — инструкция для Claude Code. Прочитай его целиком перед любой работой.

## 1. Что это за проект

Финальный проект первой недели Yessenov Data Lab 2026. Нужно собрать
**Streamlit-чатбота с RAG**, который отвечает на вопросы про гранты, программы и
деятельность фонда Шахмардана Есенова — строго по данным с сайта
`yessenovfoundation.org`.

Это учебный проект и портфолио, не заказ фонда. Цель — синтез всей недели:
интерфейс (Streamlit) + LLM (gemma4) + работа с данными (RAG).

## 2. ГЛАВНОЕ ПРАВИЛО — бот не должен выдумывать

Критерий готовности из задания: бот, который уверенно врёт про гранты —
дедлайны, требования, суммы — **не считается готовым**. По ложному ответу человек
примет реальное решение, поэтому честное «не знаю» лучше выдуманного ответа.

Реализация этого правила — несущая конструкция всего проекта:
- RAG ищет релевантные чанки; если лучший `score < RELEVANCE_THRESHOLD`, бот
  отвечает «не нашёл в материалах фонда» **без вызова LLM**.
- Системный промпт жёстко запрещает выдумывать цифры/даты/условия вне контекста.
- НЕ ослабляй эти ограничения ради «более красивых» ответов.

## 3. Архитектура

```
scrape.py        сайт фонда → обход домена → corpus.jsonl (+ процедурные карточки)
build_index.py   corpus → чанкинг → эмбеддинги text-1024 → FAISS индекс на диск
rag.py           вопрос → эмбеддинг → FAISS top-k → порог → gemma4 → ответ
app.py           Streamlit UI (чат, показ источников, лимиты, кнопка email)
email_report.py  MailerSend: саммари диалога → izok2004@gmail.com (только по кнопке)
config.py        все ключи, пути, пороги, лимиты
```

Поток данных:
1. Обходим `yessenovfoundation.org` (в пределах домена), вытаскиваем текст.
2. Режем на чанки (`CHUNK_SIZE=700`, `CHUNK_OVERLAP=120`).
3. Эмбеддим каждый чанк через `text-1024`, нормализуем L2.
4. Кладём в FAISS `IndexFlatIP` (inner product на нормированных = косинус).
5. На вопрос: эмбеддим вопрос → ищем top-4 → проверяем порог → собираем контекст
   → gemma4 с системным промптом → ответ + показ источников.

## 6. Доп. фичи (требования владельца проекта)

1. **RAG — главный приоритет.** Production-качество, FAISS, честный порог.
2. **Сценарий «где результаты / как получить доступ»** (по мотивам кейса NCOC:
   если пользователь спрашивает, как получить доступ/где выйдут результаты — бот
   даёт ссылку, куда идти). Реализован НЕ хардкодом, а через `PROCEDURE_CARDS`
   в `scrape.py` — это документы в корпусе со ссылками, RAG находит их штатно.
   Редактируй ссылки/тексты карточек под реальные данные фонда.
3. **Email-репорт** администратору после ответа пользователя — см. раздел 5.
4. **Лимиты чатбота:** счётчик сообщений за сессию (`MAX_MESSAGES_PER_SESSION`)
   + анти-спам пауза между запросами (`MIN_SECONDS_BETWEEN`). Реализовать в app.py
   через `st.session_state`.

## 8. Задачи для тебя, Claude Code

### Приоритет 1 — `app.py` (Streamlit UI)
Собери чат-интерфейс, используя готовый `rag.py`:
- Инициализация `RAG()` один раз через `@st.cache_resource`.
- `st.chat_input` + история в `st.session_state.messages`.
- На каждый ответ показывай:
  - сам ответ бота;
  - **бейдж/индикатор «grounded»** (нашёл в данных) vs «не знаю» — по полю
    `result["grounded"]`;
  - **раскрывающийся блок «Источники»** с `result["hits"]`: title, source (ссылка),
    score. Это ключевой элемент демо — показывает честность бота.
- **Лимиты:** считай сообщения пользователя; при превышении
  `MAX_MESSAGES_PER_SESSION` блокируй ввод. Проверяй паузу `MIN_SECONDS_BETWEEN`
  между запросами (через `time.time()` в session_state).
- **Кнопка «Отправить саммари администратору»** в сайдбаре: вызывает
  `rag.summarize_dialog(history)` → `email_report.send_summary(...)`, показывает
  `message_id` или ошибку. Только по нажатию, не в цикле.
- Дизайн: чистый, аккуратный (средний-хороший уровень). Заголовок, краткое
  описание бота, возможно лого-эмодзи. Не перегружай.
- Обрабатывай исключения сети (alem.ai timeout) — показывай понятную ошибку,
  не роняй приложение.

## 10. Калибровка порога (важно перед демо)

`RELEVANCE_THRESHOLD = 0.35` в config.py — стартовое значение. После сборки индекса
проверь на реальных вопросах:
- Задай 3-4 вопроса, ответы на которые ТОЧНО есть в данных → запиши их `top_score`.
- Задай вопрос, которого в данных НЕТ → запиши его `top_score`.
- Поставь порог между этими группами. Бот должен уверенно отвечать на «свои»
  вопросы и честно говорить «не знаю» на чужие. Это и проверяют на демо
  (преподаватель задаёт общий вопрос всем + ты показываешь, где бот «ломается»).
